### 创建空对象

空对象也区分两种类型，一种是有原型的空对象，一种是没有原型的空对象。

```js filename="example.js"
const obj1 = Object.create(null);
// {} -  No properties

const obj2 = {}; // or new Object()
// {} - [[Prototype]]: Object 原型是Object.prototype
```

### `Object.create(null)` 创建的空对象

`Object.create(null)` 创建的空对象通常被称为 **“纯净对象”** 或 **“字典对象”**，它与普通 JavaScript 对象有一些重要的区别，这些区别使其在特定情况下非常有用。

以下是一些 `Object.create(null)` 空对象的主要特点和用途：

- **没有原型链：** 与普通对象不同，使用 `Object.create(null)` 创建的对象没有原型链。这意味着它不继承任何属性或方法，也不包含内置的 JavaScript 对象方法，如 `toString`、`hasOwnProperty` 等。这可以防止意外的属性冲突和干扰。

- **属性访问更快：** 因为没有原型链，所以属性访问更快。在普通对象中，属性查找可能需要沿着原型链进行多次查找，而在纯净对象中，属性查找是直接的。

- **用作字典对象：** 由于它没有继承任何属性，纯净对象常被用作字典对象，用于存储键值对。在这种情况下，它不会受到原型链的干扰，并且可以更高效地存储和检索键值对。

- **防止属性冲突：** 在一些特殊情况下，您可能希望创建一个对象，其中属性的命名不受内置属性的限制，因为纯净对象不包含任何内置属性。
